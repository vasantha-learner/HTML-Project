HTML Versions & History

HTML (HyperText Markup Language) is the standard markup language for creating web pages. It has evolved significantly since its inception:

  • HTML 1.0 (1991)

    The very first version, created by Tim Berners-Lee. It supported basic text formatting, headings, paragraphs, lists, and links. There was no support for images, tables, scripts, or multimedia. Mainly used to display static documents.

  • HTML 2.0 (1995)

    Standardized by IETF. Introduced more structure and interactive features, such as forms, tables, text alignment, and hyperlinks. It became the baseline for all subsequent HTML versions.

  • HTML 3.2 (1997)

    Standardized by W3C. Added:

    • Support for scripting (JavaScript introduction)
    • Applet integration
    • Complex tables and layouts
    • New text formatting tags like <sup> and <sub>

  • HTML 4.0 (1997)

    Divided into three flavors:

    • Strict - Deprecated outdated tags, focus on CSS for styling
    • Transitional - Allowed some presentational tags for backward compatibility
    • Frameset - For creating frames-based layouts
    Introduced accessibility features, better form controls, and CSS integration.

  • HTML 4.01 (1999)

    Refined HTML 4.0, fixed errors, and clarified specifications. Fully integrated with CSS 1.0 and scripting. Marked the move toward separation of content and presentation.

  • XHTML 1.0 (2000)

    An XML-based reformulation of HTML 4.01. Stricter syntax rules:

    • All tags must be closed
    • Elements must be properly nested
    • Attribute values must be quoted
    XHTML aimed to improve consistency, compatibility, and machine readability.

  • HTML5 (2014)

    The modern standard. Key features include:

    • Semantic tags: <header>, <footer>, <nav>, <section>, <article>
    • Multimedia support: <audio>, <video>
    • Canvas <canvas> for graphics and animations
    • Local storage <localStorage> and session storage
    • Form enhancements: new input types and attributes
    • Offline support with Application Cache
    • Better accessibility and mobile responsiveness
    HTML5 is the foundation for modern web applications and interactive web pages.

  • Deprecated / Obsolete Features

    Over time, many HTML tags became deprecated or obsolete:

    • <font>, <center>, <applet>, <marquee>
    • Frames <frame> and <frameset>
    • Presentational attributes like bgcolor, align, and border
    Modern HTML encourages using CSS and semantic markup instead.

  • Milestones & Future

    HTML continues to evolve with HTML5.1 and HTML5.2 adding minor improvements. The focus remains on:

    • Better semantic structure
    • Improved multimedia and graphics support
    • Enhanced accessibility and mobile support
    • Integration with modern APIs like WebRTC, WebGL, and Geolocation

Basic HTML Tags

Basic HTML tags are the building blocks of any web page. They define the structure and content. Below is a detailed summary of each tag:

  • <!DOCTYPE html>
    Definition: Declares the HTML version used.
    Purpose: Helps the browser render the page correctly.
    Example: <!DOCTYPE html>

  • <html> ... </html>
    Definition: Root element of an HTML document.
    Purpose: Contains all HTML content for the page.
    Example: <html> ... </html>

  • <head> ... </head>
    Definition: Contains meta-information about the document.
    Purpose: Used to include page title, scripts, styles, and metadata.
    Example:
    <head>
      <title>Page Title</title>
    </head>

  • <title> ... </title>
    Definition: Specifies the title of the web page.
    Purpose: Displayed in browser tab and search engines.
    Example: <title>My Page</title>

  • <body> ... </body>
    Definition: Contains all visible content of a web page.
    Purpose: Holds text, images, tables, links, and other elements.
    Example: <body>Hello World</body>

  • <h1> ... <h6>
    Definition: Heading tags. h1 largest, h6 smallest.
    Purpose: Structure content hierarchically for readability and SEO.
    Example: <h1>Main Heading</h1>

  • <p> ... </p>
    Definition: Paragraph tag.
    Purpose: Organizes text into separate paragraphs.
    Example: <p>This is a paragraph.</p>

  • <br>
    Definition: Line break.
    Purpose: Moves content to a new line without creating a new paragraph.
    Example: Line1<br>Line2

  • <hr>
    Definition: Horizontal rule / line.
    Purpose: Separates content visually.
    Example: <hr>

  • <a href="URL"> ... </a>
    Definition: Anchor tag / hyperlink.
    Purpose: Navigate to another page or section.
    Example: <a href="https://example.com">Visit Example</a>

  • <ul>, <ol>, <li>
    Definition: List tags. ul unordered, ol ordered, li list item.
    Purpose: Organize information in bullet or numbered lists.
    Example:
    <ul>
      <li>Item 1</li>
      <li>Item 2</li>
    </ul>

  • <img src="URL" alt="text">
    Definition: Image tag.
    Purpose: Displays images on a page.
    Example: <img src="image.png" alt="Sample Image">

  • <table>, <tr>, <th>, <td>
    Definition: Table structure tags.
    Purpose: Organize data in rows and columns.
    Example: <table border="1">
      <tr><th>Name</th><th>Age</th></tr>
      <tr><td>John</td><td>25</td></tr>
    </table>

  • <div> ... </div>
    Definition: Division / block container.
    Purpose: Group content for layout or styling purposes.
    Example: <div>This is a block.</div>

  • <span> ... </span>
    Definition: Inline container.
    Purpose: Apply styles or group text inline.
    Example: <span>Highlighted text</span>

These basic tags provide the foundation for all other HTML elements. Understanding them is essential for building any web page.

HTML Text Formatting Tags

Text formatting tags are used to define the style and importance of text within a document. They control how text appears and how screen readers or search engines interpret it. Below is a detailed summary of each tag:

  • <strong> ... </strong>
    Definition: Defines **strong importance** (semantic emphasis).
    Purpose: Visually renders as **bold** by default, but its main function is semantic to indicate importance.
    Example: This is <strong>important</strong> text.

  • <em> ... </em>
    Definition: Defines **emphasis** (semantic emphasis).
    Purpose: Visually renders as *italic* by default. Used for stress emphasis, like a word or phrase that would be spoken differently.
    Example: This is <em>emphasized</em> text.

  • <b> ... </b>
    Definition: Bold text (presentational).
    Purpose: Draws attention to the text without conveying extra importance (e.g., product names or keywords).
    Example: This is <b>bold</b> text.

  • <i> ... </i>
    Definition: Italic text (presentational).
    Purpose: Indicates an alternative voice or mood, such as a technical term, a foreign phrase, or a thought.
    Example: This is <i>italic</i> text.

  • <mark> ... </mark>
    Definition: Marked / highlighted text.
    Purpose: Used to highlight or mark text that is of particular relevance in the current context.
    Example: Find the <mark>best price</mark> now.

  • <del> ... </del>
    Definition: Deleted (strikethrough) text.
    Purpose: Represents content that has been deleted from the document (often shown with a line through it).
    Example: Price: <del>$100</del> $75

  • <ins> ... </ins>
    Definition: Inserted (underlined) text.
    Purpose: Represents content that has been inserted into the document (often shown with an underline).
    Example: The change <ins>was approved</ins> today.

  • <sub> ... </sub>
    Definition: Subscript text.
    Purpose: Used for chemical formulas or mathematical variables placed below the normal line.
    Example: H<sub>2</sub>O

  • <sup> ... </sup>
    Definition: Superscript text.
    Purpose: Used for footnotes, exponents, or ordinals placed above the normal line.
    Example: E = mc<sup>2</sup>

  • <small> ... </small>
    Definition: Small print text.
    Purpose: Used for side comments, copyright, or fine print (reducing the font size one level).
    Example: <small>Terms and Conditions Apply.</small>

HTML Table Tags

HTML table tags are used to define data in a grid structure of rows and columns. Modern HTML5 emphasizes using semantic tags (like , , ) for accessibility and separating content from presentation.

  • <table> ... </table>
    Definition: The main container for all tabular data.
    Purpose: Marks the beginning and end of a data table.
    Example: <table>...</table>

  • <caption> ... </caption>
    Definition: A title or summary for the table.
    Purpose: Provides a brief description of the table's content, which is important for accessibility.
    Example: <caption>Monthly Sales Data</caption>

  • <tr> ... </tr>
    Definition: Table Row.
    Purpose: Defines a single row of cells in a table. It is the immediate child of , , , or .
    Example:<tr>...</tr>
  • <th> ... </th>
    Definition: Table Header Cell.
    Purpose: Defines a cell that contains header information (column or row labels). Text is typically bold and centered by default.
    Example: <th scope="col">Product Name</th>

  • <td> ... </td>
    Definition: Table Data Cell.
    Purpose: Defines a standard data cell within a table. This is the content of the table.
    Example: <td>Laptop Pro</td>

  • Semantic Grouping Tags (Best Practice for HTML5):

  • <thead> ... </thead>
    Definition: Table Head.
    Purpose: Groups the set of introductory rows (usually containing column headers) at the top of a table.
    Example: <thead><tr><th>...</th></tr></thead>

  • <tbody> ... </tbody>
    Definition: Table Body.
    Purpose: Groups the primary content rows of the table. This is the main data section.
    Example: <tbody>...data rows...</tbody>

  • <tfoot> ... </tfoot>
    Definition: Table Foot.
    Purpose: Groups the summary rows (like totals or footnotes) at the bottom of a table. It must come after
    • in HTML5.
    Example:<tfoot><tr><td>Total</td></tr></tfoot>

Marquee Tag

Definition:

The <marquee> tag is used to create scrolling text or images horizontally or vertically across a web page. It is a non-standard HTML tag, deprecated in modern HTML, but still supported in many browsers.

Purpose / Usage:

It is used to attract attention by animating text or images across the screen. Common uses include scrolling announcements, advertisements, or highlights.

Common Attributes:

  • direction - Specifies the scrolling direction. Values: left, right, up, down. Example: <marquee direction="left">
  • behavior -Defines the scrolling type. Values: scroll (default), slide, alternate. Example: <marquee behavior="alternate">
  • scrollamount - Controls the speed of scrolling. Higher numbers move faster. Example: <marquee scrollamount="5">
  • loop -Number of times the marquee scrolls. Default is infinite. Example: <marquee loop="3">
  • width / height - Sets the marquee size. Example: <marquee width="300">
